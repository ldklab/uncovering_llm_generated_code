The `unicode-canonical-property-names-ecmascript` package provides a set of canonical Unicode property names that are compatible with ECMAScript's regular expression Unicode property escapes. This package helps determine whether a given Unicode property name is officially recognized according to the ECMAScript standards.

### Core Functionality

1. **Exported Set of Property Names**: The package exports a set of Unicode property names that are supported by ECMAScript for use in regular expressions.

2. **API Method - `has()`**: The main exposed functionality is a method `has()`. This method checks if a given Unicode property name is part of the supported set.
   - Input: A string representing a Unicode property name.
   - Output: A boolean indicating whether the given property name is a canonical property name in ECMAScript.

3. **Installation and Usage**: Users can install the package via npm and use the `has` method to verify the validity of Unicode property names against the canonical set.

### Example Usage

- Check whether `'ID_Start'` is a valid Unicode property name: `properties.has('ID_Start')` returns `true`.
- Check whether `'IDS'` is a valid Unicode property name: `properties.has('IDS')` returns `false`.

### Publication Process for Maintainers

- Maintainers can publish new versions by bumping the version in `package.json` and pushing changes along with tags for CI/CD to handle publication.
