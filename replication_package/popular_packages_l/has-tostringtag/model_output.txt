The `has-tostringtag` package is a simple utility designed to determine if the JavaScript environment supports the `Symbol.toStringTag` feature. This feature, introduced with ECMAScript 2015 (ES6), allows objects to change the result of the `Object.prototype.toString()` method.

The package provides two main functionalities:
1. **Native Support Check**: The primary function checks if the environment has native support for `Symbol.toStringTag`. It returns `true` if this feature is present and not polyfillable, meaning it is natively supported by the JavaScript engine without any help from external libraries or shims.

2. **Sham Support Check**: A secondary exported function (`shams`) checks if there exists a sham for `Symbol.toStringTag` that mostly follows the ECMAScript specification. Shams are partial implementations to simulate native behavior using JavaScript code, often made available through libraries like `core-js` or `get-own-property-symbols`.

To implement the described functionality, the package exports two methods:
- `hasSymbolToStringTag`: Checks for native `Symbol.toStringTag` support.
- `hasSymbolToStringTagSham`: Checks for a workable sham or polyfill presence.

Here's how you can implement the `has-tostringtag` package based on the description:

```markdown
// File: lib/index.js
'use strict';

var origSymbol = typeof Symbol !== 'undefined' ? Symbol : null;

function hasNativeToStringTag() {
  return typeof origSymbol === 'function' && typeof origSymbol.toStringTag === 'symbol';
}

module.exports = hasNativeToStringTag;

// File: lib/shams.js
'use strict';

// A sham check for Symbol.toStringTag. Normally we'd inspect specific polyfills.
function hasSymbolToStringTagSham() {
  // Check for typical shams like polyfills provided by core-js or get-own-property-symbols
  return (typeof Symbol === 'function' && typeof Symbol() === 'symbol') && !!Object.getOwnPropertySymbols;
}

module.exports = hasSymbolToStringTagSham;
```
