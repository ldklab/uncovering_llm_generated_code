```javascript
// install-material-ui.js
const { exec } = require('child_process');

exec('npm install @mui/material @emotion/react @emotion/styled', (error, stdout, stderr) => {
    if (error) {
        console.error(`Error installing Material UI: ${error.message}`);
        return;
    }
    if (stderr) {
        console.error(`Installation stderr: ${stderr}`);
        return;
    }
    console.log(`Installation stdout: ${stdout}`);
});
```

```javascript
// open-docs.js
const open = require('open');

open('https://mui.com/material-ui/')
    .then(() => console.log('Material UI documentation opened in browser.'))
    .catch((error) => console.error(`Error opening documentation: ${error.message}`));
```

Explanation: This Node.js project includes two main functionalities. It installs Material UI packages and opens the Material UI documentation in a web browser. The `package.json` file defines two custom scripts: `install-material-ui` and `open-docs`. 
1. `install-material-ui.js`: Uses Node's `child_process` module to execute a shell command to install the Material UI and related packages via npm. It checks for errors and logs either the error message or, if no error occurs, the standard output.
2. `open-docs.js`: Uses the `open` package to open the Material UI documentation URL in the default web browser. It logs a success message or an error message in case of a failure.
