## Explanation

The `is-number` package provides a utility function that determines whether a given value is a finite number. It handles various edge cases commonly encountered in JavaScript, where a straightforward evaluation might yield unexpected results. The package is particularly useful for validating numeric values derived from user inputs or parsed from strings.

### Core Functionality

- **True Conditions**: The function returns `true` for values like integers, floats, valid numeric strings, and hexadecimal numbers. Examples: `5e3`, `0xff`, `'-1.1'`, `'5e3'`.
- **False Conditions**: It returns `false` for non-numeric values such as `NaN`, `Infinity`, `null`, `undefined`, strings that do not represent numbers, arrays, objects, and functions. Examples: `Infinity`, `NaN`, `'foo'`, `[]`.

The logic behind determining if a value is a number is achieved through evaluating whether a value can be parsed into a finite number using `Number.isFinite` and `parseFloat`. The library aims to be performant by optimizing these checks, and, as highlighted in benchmark results, it performs efficiently compared to alternatives.

### Implementation Code

```markdown
// is-number/index.js
'use strict';

/**
 * Check if a value is a number.
 *
 * @param {*} value - The value to check.
 * @return {boolean} - Returns true if the value is a finite number.
 */
function isNumber(value) {
  if (typeof value === 'number' && Number.isFinite(value)) {
    return true;
  }

  if (typeof value === 'string' && value.trim() !== '') {
    return Number.isFinite ? Number.isFinite(+value) : isFinite(+value);
  }

  return false;
}

module.exports = isNumber;
```
