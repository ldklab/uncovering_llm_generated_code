The `is-generator-function` Node.js package is designed to determine whether a given input is a native generator function. Generator functions in JavaScript are functions that can pause execution and resume it later, allowing you to work with asynchronous operations more easily.

### Functionality:

- The package exports a function named `isGeneratorFunction`.
- This function takes a single argument and returns a boolean:
  - `true` if the argument is a native generator function.
  - `false` if the argument is anything else (such as a regular function, `null`, etc.).
- A generator function is defined using the `function*` syntax in JavaScript, and it typically includes the `yield` keyword to pause execution.

### Usage Example:

- `isGeneratorFunction(function() {})` will return `false` because a regular function is not a generator function.
- `isGeneratorFunction(null)` will return `false` as `null` is not a function.
- `isGeneratorFunction(function* () { yield 42; return Infinity; })` will return `true` because it is a generator function.

Here's the implementation code for this functionality:

```markdown
// index.js
function isGeneratorFunction(fn) {
  if (typeof fn !== 'function') {
    return false;
  }
  const constructor = fn.constructor;
  if (!constructor) {
    return false;
  }
  // Check if function is a generator function using its constructor
  if (constructor.name === 'GeneratorFunction' || constructor === (function*(){}).constructor) {
    return true;
  }
  return false;
}

module.exports = isGeneratorFunction;

// test.js
const assert = require('assert');
const isGeneratorFunction = require('./index');

assert(!isGeneratorFunction(function () {}));
assert(!isGeneratorFunction(null));
assert(isGeneratorFunction(function* () { yield 42; return Infinity; }));
```
