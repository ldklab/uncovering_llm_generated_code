The "ESLint Plugin TypeScript" is a Node.js package that acts as a plugin for ESLint to provide linting rules specifically for TypeScript codebases. Here’s how it functions:

### How It Works

1. **Installation**: 
   - You need to have TypeScript and the `@typescript-eslint/parser` installed, as these are prerequisites for this plugin.
   - Install the `@typescript-eslint/eslint-plugin` using a package manager like yarn or npm.

2. **Configuration**:
   - Add `@typescript-eslint/parser` to the `parser` field of your `.eslintrc` configuration file to allow ESLint to understand TypeScript syntax.
   - Add `@typescript-eslint` to the `plugins` section of your `.eslintrc` file to use the rules provided by this plugin.
   - Define your desired TypeScript-specific linting rules under the `rules` field.

3. **Recommended Settings**:
   - You have the option to use a set of pre-defined recommended rules provided by the plugin by extending `plugin:@typescript-eslint/recommended` in your `.eslintrc`.
   - For projects that require type-checking, you can also extend `plugin:@typescript-eslint/recommended-requiring-type-checking`.

4. **Supported Rules**:
   - The plugin offers a comprehensive list of linting rules aimed at TypeScript-specific issues.
   - Some rules require type information, and there are markers indicating which rules are recommended, fixable, or require type information.

5. **Extension Rules**:
   - The plugin also provides extension rules for existing ESLint rules that do not support TypeScript syntax by default.

6. **Usage**:
   - Commands should include `--ext .js,.ts` to ensure ESLint processes both JavaScript and TypeScript files.

7. **Pro Tips**:
   - For performance efficiency, it’s advisable to split linting into two stages if dealing with a large codebase: syntax-focused rules (without type-checking) and type-checking rules.
