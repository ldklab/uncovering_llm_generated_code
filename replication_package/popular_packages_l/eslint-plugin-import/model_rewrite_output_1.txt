The Node.js project provided is a sample setup for using `eslint` with the `eslint-plugin-import` to manage and lint JavaScript imports, ensuring code quality and consistency in import/export practices across JavaScript files. 

### Functionality Explanation:

1. **package.json**: This file outlines the project details and manages dependencies for the Node.js project.
   - It specifies `eslint` as a development dependency, alongside `eslint-plugin-import` to facilitate linting with additional rules for imports.
   - Defines a script `lint` to run `eslint` across all files in the project directory using the command `eslint .`.

2. **.eslintrc.js**: This file configures `eslint` rules and extends configurations to include recommended practices and rules from `eslint-plugin-import`.
   - The `extends` section includes `eslint:recommended` and `plugin:import/recommended`.
   - `settings` are configured to resolve modules with specific extensions (`.js`, `.mjs`, `.json`) using Node's module resolution.
   - The `rules` section includes rules from `eslint-plugin-import` that trigger errors if violated, such as unresolved imports, naming issues, and export/import errors.

3. **index.js**: The main JavaScript file that imports a function `example` from `example.js`.

4. **example.js**: Exports a function named `example` that logs "Example function" to the console.

5. **README.md**: Provides an overview of the project and instructions for setting up by installing dependencies.

```javascript
// File: .eslintrc.js
module.exports = {
  extends: [
    'eslint:recommended',
    'plugin:import/recommended'
  ],
  settings: {
    'import/resolver': {
      node: {
        extensions: ['.js', '.mjs', '.json']
      }
    }
  },
  rules: {
    'import/no-unresolved': 'error',
    'import/named': 'error',
    'import/namespace': 'error',
    'import/default': 'error',
    'import/export': 'error'
  }
};
```
